# Build a Dota 2 Stats Viewer — TanStack Router + Tailwind v4 + Motion

> Logic and feature reference only.

> A "Spotify Wrapped"-style Dota 2 stats viewer. Users enter their Steam ID and see a
> full breakdown of their match history, hero performance, friends, lifetime records, and rank.
> All data is public — no login required. Powered by the free [OpenDota API](https://docs.opendota.com/).

## Data Fetching — OpenDota API

The [OpenDota API](https://docs.opendota.com/) is free, no auth required.

### Steam ID → Account ID Conversion

Steam IDs come in two formats:
- **Steam ID (17 digits):** `76561198071885769`
- **Account ID (9–10 digits):** `111620041` (what OpenDota uses)

```ts
function convertSteamIdToAccountId(steamId: string): string {
  if (steamId.length <= 10) return steamId // already an account ID
  const STEAM_ID_BASE = BigInt("76561197960265728")
  return (BigInt(steamId) - STEAM_ID_BASE).toString()
}
```

### API Endpoints Used

| Endpoint | Description |
|---|---|
| `GET /players/{accountId}` | Profile, avatar, rank tier, leaderboard rank |
| `GET /players/{accountId}/wl` | Overall win/loss counts |
| `GET /players/{accountId}/heroes` | Per-hero stats (games, wins, last played) |
| `GET /players/{accountId}/peers` | Co-op friends with shared game counts |
| `GET /players/{accountId}/totals` | Lifetime sums: kills, deaths, GPM, XPM, etc. |
| `GET /players/{accountId}/recentMatches` | Last 20 matches with KDA, hero, result |
| `GET /proPlayers` | List of all professional players |
| `GET /topPlayers` | Leaderboard top players |

### API Client with Retry

```ts
// src/lib/opendota-api.ts
const BASE_URL = "https://api.opendota.com/api"

async function fetchWithRetry<T>(url: string, retries = 3): Promise<T> {
  for (let i = 0; i < retries; i++) {
    const res = await fetch(url)
    if (res.status === 404) throw new Error("Player not found")
    if (res.status === 429) throw new Error("Rate limit — try again later")
    if (!res.ok) {
      if (i === retries - 1) throw new Error(`API error: ${res.status}`)
      await new Promise(r => setTimeout(r, 1000 * (i + 1))) // exponential backoff
      continue
    }
    return res.json() as Promise<T>
  }
  throw new Error("Max retries exceeded")
}

export async function getPlayerWrappedData(steamId: string) {
  const id = convertSteamIdToAccountId(steamId)
  const [player, wl, heroes, peers, totals, recentMatches] = await Promise.all([
    fetchWithRetry<Player>(`${BASE_URL}/players/${id}`),
    fetchWithRetry<WinLoss>(`${BASE_URL}/players/${id}/wl`),
    fetchWithRetry<HeroStats[]>(`${BASE_URL}/players/${id}/heroes`),
    fetchWithRetry<Peer[]>(`${BASE_URL}/players/${id}/peers`),
    fetchWithRetry<PlayerTotal[]>(`${BASE_URL}/players/${id}/totals`),
    fetchWithRetry<Match[]>(`${BASE_URL}/players/${id}/recentMatches`),
  ])

  return {
    player,
    totalMatches: wl.win + wl.lose,
    winLoss: wl,
    heroes: heroes.slice(0, 50),   // cap at 50
    topFriends: peers.slice(0, 20), // cap at 20
    totals,
    recentMatches,
  }
}
```

## Features to Build

### 1. Home Page — Steam ID Input
- Text input accepting Steam ID (17 digits) or Account ID (9 digits)
- On submit, navigate to `/id/:steamId` via TanStack Router
- Zod validation: `z.string().min(9).max(17).regex(/^\d+$/)`

### 2. Player Overview Card
Displays from `GET /players/{id}`:
- **Avatar** from `player.profile.avatarfull`
- **Username** from `player.profile.personaname`
- **Rank** from `player.rank_tier`
  - `tier = Math.floor(rank_tier / 10)`, `stars = rank_tier % 10`
  - Rank icon URL: `https://www.opendota.com/assets/images/dota2/rank_icons/rank_icon_{tier}.png`
  - Star icon URL: `https://www.opendota.com/assets/images/dota2/rank_icons/rank_star_{stars}.png`
- **Win rate** — `winLoss.win / totalMatches * 100`
  - ≥ 55%: "Top player"
  - 45–55%: "Basic player"
  - < 45%: "Goblin player"
- **KDA** — from `totals` field sums for kills/deaths/assists
- **Total playtime** in hours from `totals[].field === "duration"`

### 3. Heroes Section
Displays from `GET /players/{id}/heroes`:
- Top 5 heroes by games played
- Hero name + image from hero mapping (see Hero Data below)
- Win rate per hero: `hero.win / hero.games * 100`
- Show games played, wins, last played date (Unix timestamp → `new Date(ts * 1000)`)

### 4. Recent Matches Section
Displays from `GET /players/{id}/recentMatches` (last 20 matches):
- Each row: hero avatar, hero name, W/L result, KDA, duration, date
- **Win detection logic:**
  ```ts
  function isWin(match: Match): boolean {
    const isRadiant = match.player_slot < 128
    return isRadiant === match.radiant_win
  }
  ```
- Format duration: `Math.floor(s/60) + ":" + (s%60).toString().padStart(2,"0")`
- Date from `match.start_time` (Unix timestamp): `new Date(start_time * 1000)`

### 5. Lifetime Stats Section
Displays from `GET /players/{id}/totals` — an array of `{ field, n, sum }` objects.

Key fields to surface:

| `field` value | Display label |
|---|---|
| `kills` | Total kills |
| `deaths` | Total deaths |
| `assists` | Total assists |
| `duration` | Total hours played |
| `gold_per_min` | Average GPM |
| `xp_per_min` | Average XPM |
| `hero_damage` | Total hero damage |
| `tower_damage` | Total tower damage |
| `hero_healing` | Total healing |
| `last_hits` | Total last hits |
| `kda` | Average KDA |

```ts
// Average = sum / n for rate stats; sum alone for totals
const getTotalValue = (totals: PlayerTotal[], field: string) =>
  totals.find(t => t.field === field)?.sum ?? 0
```

### 6. Top Friends Section
Displays from `GET /players/{id}/peers`:
- Sort by `with_games` descending
- Show avatar, name, games together, shared win rate: `with_win / with_games * 100`

### 7. Pro Players Listing
Displays from `GET /proPlayers`:
- Searchable + sortable table: name, team, country code, last match date
- Filter client-side by `personaname`, `name`, or `team_name`
- Virtualize rows if list > 100 items

## Hero Data

Heroes are mapped by ID to name and avatar. Download the mapping from OpenDota:

```
GET https://api.opendota.com/api/heroes
→ [{ id, localized_name, name, ... }, ...]
```

Store this as a static JSON file and export lookup functions:

```ts
// src/lib/heroes.ts
import heroesJson from "./heroes.json"

const heroMap = new Map(heroesJson.map(h => [h.id, h]))

export function getHeroName(id: number): string {
  return heroMap.get(id)?.localized_name ?? "Unknown Hero"
}

export function getHeroAvatar(id: number): string {
  const hero = heroMap.get(id)
  if (!hero) return "/fallback-hero.png"
  // hero.name is like "npc_dota_hero_antimage" → strip prefix → "antimage"
  const key = hero.name.replace("npc_dota_hero_", "")
  return `https://cdn.cloudflare.steamstatic.com/apps/dota2/images/heroes/${key}_full.png`
}
```

## Rank System Reference

OpenDota returns `rank_tier` as a two-digit number: `tens digit = rank, ones digit = stars`.

| rank_tier | Rank | Stars |
|---|---|---|
| 11–15 | Herald | 1–5 |
| 21–25 | Guardian | 1–5 |
| 31–35 | Crusader | 1–5 |
| 41–45 | Archon | 1–5 |
| 51–55 | Legend | 1–5 |
| 61–65 | Ancient | 1–5 |
| 71–75 | Divine | 1–5 |
| 80 | Immortal | — |

```ts
const RANK_NAMES = ["Herald", "Guardian", "Crusader", "Archon", "Legend", "Ancient", "Divine", "Immortal"]

export function getRankName(rank_tier: number): string {
  const tier = Math.floor(rank_tier / 10)
  return RANK_NAMES[tier - 1] ?? "Unranked"
}

export function getRankStars(rank_tier: number): number {
  return rank_tier % 10
}

export function getRankIconUrl(rank_tier: number): string {
  const tier = Math.floor(rank_tier / 10)
  return `https://www.opendota.com/assets/images/dota2/rank_icons/rank_icon_${tier}.png`
}
```

## Common Pitfalls

1. **Always convert Steam ID to Account ID** before calling any `/players/{id}` endpoint
2. **Win detection** uses the `player_slot` field, not a simple `win` boolean — slot < 128 is Radiant
3. **Totals are sums not averages** for count fields; divide by `n` for averages (e.g., GPM, KDA)
4. **Hero images** use the internal name (`npc_dota_hero_antimage` → `antimage_full.png`), not the display name
5. **Rank tier 0 or undefined** means unranked — guard before rendering rank icons
6. **Peers endpoint** returns all-time peers; sort by `with_games` for "friends" ranking
7. **Rate limiting** — 50ms between requests; use `Promise.all` for parallel calls in a single request cycle, not sequential loops
